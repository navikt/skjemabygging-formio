# Test cases 09-20: PDF end-to-end verification

PDF end-to-end verification using the tc-pattern. These test cases verify that the backend
generates the correct PDF request body and sends it to familie-pdf.

## tc09 - Paper identity (components form)

Verifies the PDF body when only identity fields are filled on the `components` form with paper submission.
Includes signature section with two signatures (Eier, Deleier).

## tc10 - Digital identity (components form)

Verifies the PDF body when only identity fields are filled on the `components` form with digital submission.
No signature section. Uses logged-in user's fødselsnummer (08842748500).

## tc11 - Nologin identity (components form)

Verifies the PDF body for nologin submission on the `components` form with identity only.
Includes legitimasjon section instead of signatures. Uses user-typed fødselsnummer (20905995783).

## tc12 - Digital all values (components form)

Verifies the PDF body when all component fields are filled on the `components` form with digital submission.
Covers: text fields, numbers, checkboxes, selects, radio, person fields, money/account,
organization, date/time, grouping/repeating data, and confirmation checkbox.

## tc13 - Conditional pages all (conditionalpage form, digital)

All conditional pages visible (3 checkboxes on page 1, checkbox on page 2, 2 checkboxes on page 3).

## tc14 - Conditional pages minimal (conditionalpage form, digital)

No conditional pages visible (no checkboxes checked on page 1, skips to summary).

## tc15 - Conditional pages paper (conditionalpage form, paper)

Paper submission with 1 checkbox + text field filled. Includes signature section.

## tc16 - Default signature (stpaperdigital form, paper)

Verifies the default empty signature rendering (single unnamed signature field).

## tc17 - Old default signature (stpaper form, paper)

Verifies backward-compatible signature rendering when `signatures` property is undefined.

## tc18 - Two signatures English (components form, paper, lang=en)

Verifies English translation of signature labels (Owner, Part owner) and form content.

## tc19 - Attachment with comment (components form, paper)

Verifies attachment panel rendering including comment text in the PDF.

## tc20 - Data fetcher activity (datafetchercontainer form, digital)

Verifies aktivitetsvelger selection is included in the PDF body.

## Generating new fixtures

To capture the PDF request body for a new test case:

1. Activate the capture variant: `cy.mocksUseRouteVariant('post-familie-pdf:capture')`
2. Run the Cypress test
3. Find the captured body in `/tmp/captured-pdf/captured-pdf-body-N.json`
4. Review and rename to `tcXX-descriptive-name.json`
5. Add import and route variant in `mocks/mocks/routes/familie-pdf.ts`
6. Switch test to use `cy.mocksUseRouteVariant('post-familie-pdf:success-tcXX')`
